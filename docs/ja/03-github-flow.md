# GitHub Flow

## GitHub Flow とは？

GitHub Flow は、定期的にリリースを行うチーム向けに設計された、軽量なブランチベースのワークフローです。
以下の 6 つのステップで構成されています

1. `main` から **ブランチを作成**します。
2. ブランチに **コミットを追加**します。
3. ディスカッションを開始するために **プルリクエストを開く**。
4. 共同作業者と変更内容を **レビューおよび議論**します。
5. ブランチから**デプロイ**し、本番環境で検証します（任意）。
6. Pull Requestを`main`に**マージ**します。

## ブランチの命名規則

説明的で、アンダースコアで区切られた小文字の名前を使用し、先頭にゼロを埋めた3桁の課題番号を付けてください

```text
002_add_login_page
005_broken_nav_link
007_update_readme
```

## Pull Request の作成

1. ブランチを GitHub にプッシュします

   ```bash
   git push origin your-branch-name
   ```

2. GitHub で Pull Request を作成します。
3. タイトルと説明を入力します。
4. チームメイトにレビューを依頼します。

## プルリクエストのマージ

PRが承認されたら

1. GitHubで**「プルリクエストをマージ」**をクリックします。
2. リモートブランチを削除します。
3. 最新の`main`をローカルにプルします

   ```bash
   git switch main
   git pull origin main
   ```

## 実践演習

1. 新しいブランチ `feature/my-first-pr` を作成します。
2. `docs/` ディレクトリに新しい Markdown ファイルを追加します。
3. `npm run commit` で変更をコミットします。
4. ブランチをプッシュし、GitHub でプルリクエストを作成します。
5. 仲間にレビューとマージを依頼します。
